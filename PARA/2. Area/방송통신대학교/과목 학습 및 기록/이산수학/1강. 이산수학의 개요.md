## 문제 해결을 위한 과정

<img width="557" alt="Image" src="https://github.com/user-attachments/assets/1e3cc921-2f76-4cfb-b208-a36b8c9a498c" />

- **수학적 모델링을 이용한 문제해결**
    - 문제 → 추상모델 → 변형된 모델 → 문제의 해결책
    - 실제 문제를 수학적 개념과 관계로 **추상화**해 모델을 만들고, 이를 수학적으로 처리하여 해결책을 도출하는 과정
- **정보 모델링을 이용한 문제해결**
    - 문제 → 정보 → 처리 → 문제의 해결책
    - 문제에서 필요한 정보를 추출하고 이를 처리하여 해결책을 찾는 과정

## 추상화
> 핵심만 모아놓고, 필요 없는 건 다 빼는 것

- **정의**: 일정한 인식 목표를 추구하기 위하여 여러 가지 표상(현상)이나 개념(대상)에서 특정한 특성이나 속성을 빼냄
- **과정**: 문제와 관련된 핵심 내용만 남기고 관련 없는 내용을 제거해 문제를 **단순화**시키는 과정

### 예시
#### 1. 일상 문제의 수학적 추상화
- **원래 문제** 
	과일가게 아저씨가 사과 1개를 600원에 주고 사서 800원에 팔고, 배 1개를 1200원에 주고 사서 1500원에 팔았다고 할 때, 사과와 배를 합하여 총 10개를 팔고 이익이 2400원이 생겼다고 한다면 사과는 몇 개를 팔았습니까?
- **추상화된 문제 표현**
	- Let A: 사과의 판매 개수
	- Let B: 배의 판매 개수
- **수학적 방정식으로 변환**
	- A + B = 10 (총 판매 개수)
	- 200A + 300B = 2,400 (총 이익)
#### 2. 디지털 논리 회로의 간소화

<img width="573" alt="Image" src="https://github.com/user-attachments/assets/fcb3c81b-fd53-474d-affd-48ffabe3ea45" />

- 복잡한 논리 게이트 회로가 불 대수(Boolean algebra) 표현식으로 추상화됨
- F(X, Y) = X + XY + X̄Y

이는 전자 회로의 물리적 구현에서 수학적 함수로의 추상화를 보여줌. 이 추상화를 통해 회로의 동작을 분석하고 최적화하는 것이 더 쉬워짐.

|                        |            |
| ---------------------- | ---------- |
| F(X, Y) = X + XY + X̄Y |            |
| = X(1 + Y) + X̄Y       | 분배법칙       |
| = X + X̄Y              | 1 + Y = 1  |
| = (X + X̄)(X + Y)      | 분배법칙       |
| = X + Y                | X + X̄ = 1 |
결국 `X + XY + X̄Y`가 `X + Y`의 회로와 같아짐

# 알고리즘과 표현
> - 어떠한 문제를 해결하기 위한 여러 동작들의 유한한 모임
> - **"a set of instructions**, sometimes called a procedure or a function, that is used to perform a certain task." [Wikipedia]
> - "an explicit **step-by-step** procedure for solving the problem." [Dossey, et al.]

알고리즘은 문제 해결을 위한 명확하고 체계적인 단계별 지시사항.
## 알고리즘의 표현 방법
***이거 시험 잘나옴***
### 1. 컴퓨터 프로그래밍 언어(Computer Programming Language)
- C, Java, Python 등의 프로그래밍 언어를 사용하여 알고리즘을 코드로 표현
	- 구체적이고 정확하지만 특정 언어의 문법 규칙을 따라야 함
	  => 통일된 언어가 존재하지 않음
- 컴퓨터 작동을 위한 동작을 세밀하게 지시 , 컴퓨터가 직접 실행할 수 있는 형태
- 알고리즘의 핵심요소가 잘 드러나지 않음 
- 중요하지 않은 부차적인 표현에 신경써야 함
### 2. 순서도(Flow Chart)
- 알고리즘의 흐름을 시각적으로 표현하는 도구 (도식화)
- 각 단계와 결정 사항을 **기호와 화살표**로 나타냄
- 직관적으로 이해하기 쉽지만 복잡한 알고리즘의 경우 그림이 복잡해질 수 있음
### 3. 의사코드(Pseudocode)
- 자연어와 프로그래밍 언어의 중간 형태
- 프로그래밍 언어의 엄격한 문법에 구애받지 않으면서도 알고리즘의 구조와 논리를 명확하게 표현
- 이해하기 쉽고 특정 프로그래밍 언어에 의존하지 않음
- 나중에 다양한 프로그래밍 언어로 쉽게 변환 가능
- 모호한 부분은 프로그래밍 언어의 문법을 채용하여 명확하게 기술 
- 구체적으로 표현할 필요가 없는 부분은 자연어를 통해 설명식으로 기술 
- 알고리즘의 작동방식을 설명하는 용도로만 사용 

강의에서는 C언어를 기반으로 하는 의사코드 사용함.

알고리즘을 표현하기 위해 할당문, 제어문으로 알고리즘을 나타내고, 3가지 제어 구조를 가짐

## 알고리즘 기본 제어구조

의사코드는 크게 두 부분으로 구성됨.

- **할당문** : 변수에 값을 할당하는 문장
- **제어문** : 알고리즘의 흐름을 제어하는 문장
    - **순차문** : 명령을 순차적으로 실행
    - **조건문** : 조건에 따라 실행 여부 결정
    - **반복문** : 특정 조건이나 횟수에 따라 반복 실행

### 기본 제어구조 (Control Structure)

알고리즘의 기본 제어구조는 3가지로 분류.
#### 1. 순차구조 (Sequence)
- 명령어가 위에서 아래로 순차적으로 실행되는 구조
- A → B → C 순서로 실행됨
- 가장 기본적인 제어 흐름

#### 2. 선택구조 (Selection)

- 조건에 따라 서로 다른 명령을 실행하는 구조
- 조건이 참(Y)이면 한 경로, 거짓(N)이면 다른 경로로 진행
- 구현 방법:
    - **if문**: 조건이 참일 때만 특정 코드 실행
    - **switch문**: 여러 경우에 따라 다른 코드 실행

#### 3. 반복구조 (Iteration)

- 특정 조건이 만족될 때까지 명령을 반복 실행하는 구조
- 조건 검사와 명령 실행의 반복
- 구현 방법:
    - **for문**: 정해진 횟수만큼 반복
    - **while문**: 조건이 참인 동안 계속 반복
    - **foreach문**: 컬렉션의 각 요소에 대해 반복

이러한 세 가지 기본 제어구조를 조합하여 복잡한 알고리즘을 구현할 수 있습니다. 어떤 프로그래밍 언어를 사용하든, 이 기본 구조들은 보편적으로 적용됩니다.


# 주요 용어

- **이산수학**: 이산적(불연속적)인 수학 구조를 연구하는 학문. 컴퓨터 과학의 기초가 되며, 개별적이고 구분된 값들을 다룬다.
- **데이터 모델링**: 실세계의 데이터를 컴퓨터가 이해하고 처리할 수 있는 형태로 구조화하는 과정. 데이터 간의 관계와 제약조건을 정의한다.
- **순서도**: 알고리즘이나 프로세스의 흐름을 시각적으로 표현하는 도구. 특정 기호를 사용하여 작업의 순서와 의사결정 과정을 나타낸다.
- **연속수학**: 연속적인 값과 구조를 다루는 수학 분야. 미적분학, 해석학, 위상수학 등이 포함되며, 끊김 없이 변화하는 값을 연구한다.
- **추상화**: 문제의 핵심 요소만 남기고 불필요한 세부사항을 제거하는 과정. 복잡한 문제를 단순화하여 본질에 집중할 수 있게 한다.
- **의사코드**: 알고리즘을 표현하는 방식으로, 프로그래밍 언어와 자연어의 중간 형태. 특정 프로그래밍 언어의 문법에 얽매이지 않고 알고리즘의 논리를 명확하게 기술할 수 있다.
- **수학적 모델링**: 실세계의 문제나 시스템을 수학적 언어와 개념을 사용하여 표현하는 과정. 현실 문제를 수학적으로 분석하고 해결하는 데 활용된다.
- **프로그래밍 언어**: 컴퓨터에 명령을 내리기 위한 형식화된 언어. 알고리즘을 컴퓨터가 실행할 수 있는 형태로 표현하는 데 사용된다.

1. **이산수학(Discrete mathematics)**: 이산적인 수학구조에 대해서 연구하는 학문. Discrete mathematics is the study of mathematical structures that are fundamentally discrete rather than continuous.
2. **추상화(abstraction)**: 문제와 관련된 핵심내용만 남기고 관련 없는 내용을 제거하여 문제를 단순화 시키는 과정. Abstraction is a process by which concepts are derived from the usage and classification of literal ("real" or "concrete") concepts, first principles (such as an axiom), or other methods.
3. **알고리즘(algorithm)**: 알고리즘 표현방법으로는 컴퓨터 프로그래밍 언어(computer programming language), 순서도 (flow chart), 의사코드 (pseudocode) 등의 방법이 있음. An algorithm is a set of instructions, sometimes called a procedure or a function, that is used to perform a certain task.
